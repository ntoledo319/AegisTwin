# Privacy and Git History Purge Guide

**AegisTwin Security Documentation**

---

## Overview

This document describes how PII (Personally Identifiable Information) is handled in the AegisTwin repository and provides instructions for purging sensitive data from git history.

## PII Handling Policy

### Non-Negotiables
1. **NO PII ships** - No personal messages, contact exports, raw logs, or screenshots of real data
2. **All demos use synthetic data** - Generated by `tools/synth_data_gen.py`
3. **PII must be removed from working tree AND git history**

### Quarantine Structure
```
graveyard/
├── PII/                    # Quarantined personal data (never committed)
│   ├── phoneshit1/         # Entire phoneshit repo (real conversation data)
│   ├── dt_summarized_conversations/
│   ├── messages_complete.json.zip
│   └── ...
└── RETIRED_CODE/           # Deprecated code (may be committed for reference)
```

### .gitignore Protection
The `.gitignore` file includes patterns to prevent accidental reintroduction:
- `graveyard/PII/`
- `*messages*.csv`, `*conversation*.txt`
- `*jessica*`, `*contact*.json`
- `*.sqlite`, `*.db`

## Scanning for PII

Run the PII scanner before any release:

```bash
python tools/pii_scan.py
```

The scanner will:
1. Check filenames against suspicious patterns
2. Scan file contents for emails, phone numbers, names
3. Check SQLite databases for suspicious tables
4. Generate a report at `docs/PII_SCAN_REPORT.md`
5. Exit with code 1 if any PII is found outside `/graveyard/`

## Purging Git History

If PII was previously committed to git history, it must be permanently removed.

### Prerequisites
Install git-filter-repo:
```bash
pip install git-filter-repo
# or
brew install git-filter-repo
```

### Running the Purge

```bash
chmod +x tools/purge_history.sh
./tools/purge_history.sh
```

The script will:
1. Create a backup tag (`pre-purge-backup`)
2. Remove all paths listed in the script from ALL commits
3. Remove files matching PII patterns from history

### Manual Purge (Alternative)

If you need to remove specific files manually:

```bash
# Remove a specific path from all history
git filter-repo --path path/to/sensitive/file --invert-paths

# Remove files matching a pattern
git filter-repo --path-glob '*sensitive_pattern*' --invert-paths

# Remove by regex
git filter-repo --path-regex '.*\\.sqlite$' --invert-paths
```

### After Purging

1. **Force push** (if repo is on a remote):
   ```bash
   git push origin --force --all
   git push origin --force --tags
   ```

2. **Notify collaborators** - They must re-clone the repository

3. **Re-run PII scanner** to verify:
   ```bash
   python tools/pii_scan.py
   ```

4. **Delete local backups** that may contain PII

## Synthetic Data

All demos use synthetic data generated by `tools/synth_data_gen.py`:

```bash
python tools/synth_data_gen.py
```

This creates fixtures in `/fixtures/`:
- `demo_small.json` - Quick demo dataset
- `demo_medium.json` - Fuller demo dataset
- Individual fixture files (contacts, messages, events, emails)

**All synthetic data is clearly marked with metadata:**
```json
{
  "metadata": {
    "is_synthetic": true,
    "warning": "This is entirely fictional data..."
  }
}
```

## Verification Checklist

Before any release or demo:

- [ ] `python tools/pii_scan.py` passes (exit code 0)
- [ ] No files in working tree match PII patterns
- [ ] `/graveyard/PII/` is in `.gitignore`
- [ ] Synthetic fixtures are generated and used
- [ ] Git history has been purged (if needed)
- [ ] `diligence_pack/PII_PURGE_REPORT.md` is up to date
